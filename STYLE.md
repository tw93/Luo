# Luo 落文字体特征抽象与批量改字规范

这份文档用于给其他 AI 或后续批量改字流程学习落文风格。它描述的是气质、结构判断和批量处理边界，不是某个字的临摹稿。

## Summary

- 落文是一套“清骨、端正、可打印”的现代中文字体，不是书法复刻、手写字、石刻字或古风装饰字。
- **v0.4 风格比例：私有印刷楷意参考抽象出的理性楷意印刷字 60% / 魏晋小楷骨法 30% / 现代字体工程稳定性 10%。**（v0.3 比例 50/40/10 见 [HANDOFF.v0.3.md](HANDOFF.v0.3.md)）
- 核心目标：字面饱满、骨架稳定、笔画决断、收笔干净，长文排版安静，小字号打印清楚。
- 其中“理性楷意印刷字”指 private typographic-kai reference 所抽象出的现代字库化楷意：字面开阔、笔画明确接近直线、转折有清晰骨节、钩短利、撇捺有速度但少弯弧；不再像 LXGW WenKai 那样保留手写软楷的弯曲感。
- 当前主参考固定为 TsangerJinKai02-W04；TsangerJinKai02-W05 只做黑度上限检查，不作为继续追黑、追重的目标。
- 西湖石刻只提供局部提醒：短、涩、利、收得住；不得引入破损、噪点、纹理、行书连带。

## 理性楷意印刷字特征抽象

这类字体不是传统楷书复刻，也不是手写字，而是一种经过现代字库工程理性化的楷意字体。它保留楷书横、竖、撇、捺、钩、折的方向感，但笔画轮廓被整理成接近直线段，转折由清晰的骨节连接，结构更适合屏幕和印刷阅读。

字面偏大，整体舒展，不做窄体；横向空间打开，左右部件不拥挤。字形有明显骨架感，笔画走势分明而不是手写式柔顺。它的气质不是古典碑帖的厚重，也不是手写文楷的松软，而是清爽、现代、决断、可读。

横画接近直线，可有 1 度内的极轻上扬，但不能保留 LXGW 的明显弯弧。竖画承担主要骨架，略重于横画，整体接近直线，不能形成手写式的弯曲。撇捺有速度感，起笔和收笔干净；中段不再保留 LXGW 那种"软"的弧度，向 commercial print-kai reference 的"线段感"靠。点画短促、有方向，不圆、不糊、不水滴化。钩画明显偏短，收束更利落。

转折处有清晰骨节，转折处略加重，外圆内锐；不能再像 LXGW 那样圆滑过渡。端点保持软切角，但更靠近 private typographic-kai reference 的扁平切口；横端、竖底、撇捺端、点端、钩端分开判断。

这类字体的关键是“楷意被排版化”：看起来有书写来源，但已经不是手写；有传统骨法，但服务于现代排版，更接近活字而非毛笔。它适合标题、说明文字和中短篇阅读，气质决断、舒展、干净，有文化感但不古风化。

## Style DNA

- 骨架：端正舒展，不做窄体；重心稳，字内空间均匀；简单字不虚，复杂字不糊。
- 灰度：整体黑度中等偏清，竖画承担骨架，横画略轻，撇捺有速度但不飞。
- 横竖：横画接近直线、可有 ≤1 度极轻上扬，长横稳、短横有顿挫；竖画稳直、不再保留手写式弯曲。LXGW 风格的明显弯弧必须由 `straighten_strokes` pass 拉平。
- 点画：短促、有方向、小楷楔形感；不圆点、不水滴、不斜杠。氵、讠、心、雨字头等点画只做角度归一和轻微形态整理。
- 钩画：短、准、清、利；钩根不肿，钩尾不甩。v0.4.8 精修后 `HOOK_FINAL_SHORTEN=0.16`、`HOOK_FINAL_TIP_SHARPEN=0.13`，弯钩/竖弯钩（角度 ≥80°）走更轻的 `HOOK_FINAL_TIP_SHARPEN_CURVED=0.08` —— 只对 tip on-curve 做轻拉，off-curve 句柄保留垂直位置，钩尾继续读为连续曲线，不再像两段折线相接。
- 转折：有清晰骨节，外圆内锐。v0.4 全 CJK 跑骨节修整保留；默认 `TURN_FINAL_DISPLACE=1.6`（正文不再制造尖折点），只对 `TURN_FINAL_CHARS` 锚字组保留 `TURN_FINAL_PRIORITY_DISPLACE=1.75` 的骨节，标题/锚字仍能读出骨节但不显钉头。
- 端点：软切角，横端以 Tsanger W04 式干净角切为主，不再新增长横末端顿笔；`luo_horiz_cap_flatten` 只清理 cap 微台阶，不制造新顿挫。不尖刺、不圆熟、不刀切。端点要有统一气质，但不能有统一形状；横端、竖底、撇捺端、点端、钩端分开判断。
- 撇捺：撇尾细而不尖，捺脚不圆胖；简单字可舒展，复杂字要收住。
- 复杂字：优先打开内部白，副笔通常减轻 3% 到 5%，雨字头、艹头、鹿、鬲、赢、魔等密集部件可局部放到 5% 到 8%；主笔保留力度，不能靠堆黑显得有力。
- 部件层级：学习 private typographic-kai reference 时优先学“主副关系”，不是只学直线。氵旁、讠旁、艹头、密字内白都要有层级；`清/源/落` 的左旁不能和主体抢黑，`言/试/诸/话` 的点与副横要短而轻，`实/寒` 下盘要收，`远/造/这` 的走之底不能拖黑，`皆/熹/覆/藏` 要开内白、拉层距，并轻收上部密集部件，`斗/考/兴/今` 这类宽字和斜势字只收边缘与下尾，`书` 的右点要短促收住。
- 排版：CJK 字距保持 1em 节奏，标点可略窄；长文灰度稳定，横排竖排都成立。
- 英文和数字搭配：优先选择 humanist、轻调制、不过度几何的西文字体；小字号优先基线和 x-height 稳定，纸面气质由中文 Luo 承担；避免过硬的科技感无衬线，也避免太古典的高对比衬线。
- 定位边界：Luo 是独立的纸面阅读中文字体；外部产品或项目只能作为设计语境和鸣谢来源，不能写成 Luo 的用途边界。
- Latin 边界：`Luo Regular` 保持 CJK-only，不内置 ASCII；英文和数字走推荐 fallback。未来如果做内置 Latin，应作为可选独立版本，不覆盖主版。

## Batch Editing Rules

- 先判断 glyph 类型，再处理：点类、钩类、转折类、多横类、框形类、复杂密集类、撇捺舒展类要分通道处理。
- v0.3 只做 starter 子集收口：官网、README、纸面试读、内部 A4 打印样张、核心锚字和 GB2312 校准页体验。不要在 v0.3 final 里启动大范围扩字。
- v0.4 扩字先从 GB2312 一级常用字开始，再进入完整 GB2312 实验构建；扩字按部件和结构通道批量扩散，不按单字硬补。
- 禁止一刀切固定比例。对输入差异大的属性使用自适应 lerp，例如点画长短轴、复杂字内部收放、局部 bolden 限制。
- 点类 contour 的 bolden 只能限制横向过度膨胀，不能压掉纵向厚度，避免 `代黑点述游清流` 这类斜点/短撇变细。
- SOFT 点画通道语义固定为“角度归一，轻碰形态”，不能继续压短成长斜杠或圆点。
- 钩画只做最终尾部收束，不恢复全局长钩，不增加钩重。弯钩/竖弯钩 (角度 ≥80°) 走 `HOOK_FINAL_TIP_SHARPEN_CURVED` 软通道；不要把这个软通道值再升回来，否则会重现 v0.4 的两段折线。
- 转折在 v0.4 已覆盖所有 CJK，但仍需保持轻触；正文默认 `TURN_FINAL_DISPLACE=1.6`，只有 `TURN_FINAL_CHARS` 锚字组吃 `TURN_FINAL_PRIORITY_DISPLACE=1.75`。误伤点画、低点数 contour 或框形字时先调局部识别和 frame 参数，不回退到圆滑过渡，更不要把全局 displace 拉回 2.5+。
- 复杂字只开内部白，不动整体字宽，不改 CJK advance。
- 不通过整体加粗、整体收窄、整体放大字面来解决局部问题。
- 已覆盖字的 all-covered identity pass 只能作为护栏，轻触字面、内白和端点方向，不能全量强推结构。真正的去源字体相似度必须按白名单分批做结构、内白、主副笔和端点语法重写。
- 当前 `identity_core_v2` 只允许三类白名单结构重写：框形字开内白和稳外框，上下多层字拉层距并减副横，撇捺/长斜字增加张力和尾部收束。不要把这套参数扩大到所有已覆盖字。
- 结构白名单要按整组验收：框形字先看内部白和外框稳定，上下多层字先看中宫透气和副横层级；如果某个字因同类参数更像源字体，宁可暂时保留 guardrail，也不要硬改。
- `月` 是 v0.4.8 冻结锚字：最终轮廓必须与 Luo v0.3 baseline 完全同点位。它不能再被通用 frame/grid pass 重新解释；局部模拟 v0.3 会混出不自然的左扫笔和右下钩节奏。
- `答` 只修底部口框下边存在感，不全字加粗，也不撑大内白。它是 final visible-defect correction，不代表所有上下结构都应该补底。
- Tsanger W04 是当前唯一主参考；W05 只看是否超过黑度上限，不把 W05 的更重状态当成下一轮方向。
- `HOMEPAGE_P6_LEFT_RIGHT_GLYPHS` 是 homepage/starter 当前可见队列，不是 GB2312 扩字入口。左右结构风险用 `luo_homepage_p6_left_right` 的共享 topology refiner 处理：左旁降灰、中宫开白、右下尾轻收。
- `luo_horiz_cap_flatten` 和 `luo_diag_endpoint_clean` 是目标队列 final polish，只服务当前截图风险字；不要全局化，也不要把它们扩大成新的大覆盖 pass。
- `luo_curve_tail_polish` 只处理截图反馈的 `风气成` 右下曲尾，目标是连续、短、收住；不要用它替代全局 hook/tail 规则。`荒` / `自` 的单字修补也只属于 final visible-defect correction，不代表 dense-top 或 frame 字可以整体加码。
- 凡是让字变得更亲和、更圆润、更手写、更舒展的处理，都要二次审查；凡是让字更清、更稳、更有骨节但不变粗的处理，优先保留。
- v0.4 加入 `straighten_strokes` 后，目测看 bow distance 应该从 v0.3 的 ~29u 降到更接近私有印刷楷意参考的区间。这是去源字体相似度的真正杠杆。
- 官网优先字不能只是“更有骨的源字体”。但 256x256 raster IoU 对纯权重变化不敏感（笔画变细时反而 IoU 升高），所以 v0.4 不再把 IoU 当硬门，改用「IoU + bow distance + 视觉对照」三位一体判断。普通字 IoU 0.60 是历史目标，留作回归参考；私有参考门只写入 `local/ref/metrics/` 的本地报告。
- `straighten_strokes` 有 STRAIGHTEN_SKIP_CHARS 名单豁免心字底/走之底等已有专门 pass 的字；扩字时如果发现新的部件被误伤，把字加进豁免名单是首选，不要降低 STRAIGHTEN_BLEND 全局值。
- `proof/gb2312.html` 只作为覆盖检查页，公开状态保持 `已覆盖` / `待补字` 两类；结构优化分类只用于工程回归，不作为页面判断标准。

## AI Prompt Template

正向提示：

```text
请将这些中文 glyph 调整为 Luo v0.4 落文风格：决断、端正、有骨节、可打印；横画接近直线、有 ≤1 度极轻上扬，竖画稳直，撇捺速度感强但中段不弯弧，钩画短而利落，转折有清晰骨节，端点软切但偏扁平，复杂字内部透气，整体安静耐读。

气质参考私有印刷楷意参考的开阔骨架与决断笔形，以及魏晋小楷的清瘦骨法。字面偏大，横向舒展，轮廓干净，笔画有书写方向但已经字体化、印刷化。最终必须保持现代字体轮廓稳定、干净、可打印。
```

反向提示：

```text
不要做成 LXGW WenKai 那种圆润手写软楷，不要保留笔画中段的明显弯弧。不要做成行书、草写、手写文楷、碑帖复刻、石刻复刻、景区导视字、古风装饰字。不要加入破损、噪点、飞白、墨迹、纹理。不要让字变圆、变胖、变软、变窄、变粗糙。不要拉长钩尾，不要把点画做成圆点、水滴或斜杠。不要复制任何现成商业字库或私有参考的具体轮廓，只学习“理性楷意、决断骨节、清瘦舒展、可读印刷”的抽象气质。
```

## Acceptance Criteria

- 大字看：`落文` 要清骨舒展、决断、有骨节，不能松软、圆胖或古风。v0.4 比 v0.3 应该明显更"硬"。
- 标题看：`以文为名，以字落纸` 要有书卷气，但不手写化。撇捺中段不弯。
- 小样看：`代黑点述游清流` 的斜点/短撇不能偏细、不能斜杠化。
- 长文看：《兰亭集序》连续阅读灰度安静，重复字不跳，复杂字不堵。
- 打印看：10pt、12pt、14pt、16pt、24pt、48pt 黑白输出都清楚。v0.4.2 把 BOLDEN 回到 v0.3 baseline 6/14（v0.4 试过 4/11 太薄、5/13 配合 v0.4.1 系统回退后还是欠灰度），配合 straighten 的角点保护，10pt 不再发虚也不糊。
- 回归底线：仍然是落文，不是任何现成字库复刻；不是 LXGW WenKai 软楷，不是商业字库抄轮廓，不是石刻字。
- 私有参考门：设置 `LUO_PRIVATE_KAI_REF` 后，本地 `local/ref/metrics/` 报告可用于检查 private reference band；`lxgw_below_max` 是软指标（IoU 对纯权重不敏感）。
- Tsanger W04 raw 应保持在 `0.56-0.60`；W05 只作上限检查，不能因为追 W05 让正文黑度继续上升。
- 全站 `raw >= 0.87` 风险字应持续下降，且 generic bucket 不能回升；如果两者冲突，优先保住 generic 均值和 body 可读性。
